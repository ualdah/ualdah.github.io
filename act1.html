1. Tipos de navegación

En Ionic, tenemos distintas alternativas para realizar navegación entre páginas. En las siguientes subsecciones se describen los principales mecanismos.

1.1. NavController

Para analizar el comportamiento del uso del NavController, creamos una aplicación Ionic vacía:

$ ionic start navApp blank --type=angular

Seleccionaremos la opción por defecto crear la aplicación haciendo uso de NgModules.

Ejecutamos la aplicación para ver su comportamiento:

$ cd navApp
$ ionic serve

El siguiente paso es crear una nueva página a la que poder acceder desde la página home creada por defecto.

Para crear una nueva página, ejecutamos el siguiente comando:

$ ionic generate page newPage
En lugar de generate, podemos escribir g únicamente. Ionic CLI, al igual que Angular CLI, ofrecen atajos para los diferentes comandos de su interfaz.

Como resultado, dentro de la carpeta new-page se crea un componente de tipo página que está formado por los sguientes archivos:

  • new-page-routing.module.ts

  • new-page.module.ts

  • new-page.page.html

  • new-page.page.scss

  • new-page.page.spec.ts

  • new-page.page.ts

Aunque el nombre introducido para el componente de tipo página era newPage, Ionic CLI lo cambia automáticamente para cumplir con las restricciones de nombre establecidas (ver documentación de Angular en el siguiente enlace).

Si abrimos el archivo app-routing.module.ts, vemos que automáticamente se ha creado una nueva entrada las direcciones o rutas gestionadas por el RouterModule:

Archivo app-routing.module.ts

import { NgModule } from '@angular/core';
import { PreloadAllModules, RouterModule, Routes } from '@angular/router';

const routes: Routes = [
  {
    path: 'home',
    loadChildren: () => import('./home/home.module').then( m => m.HomePageModule)
  },
  {
    path: '',
    redirectTo: 'home',
    pathMatch: 'full'
  },
  {
    path: 'new-page', (1)
    loadChildren: () => import('./new-page/new-page.module').then( m => m.NewPagePageModule)
  },
];

@NgModule({
  imports: [
    RouterModule.forRoot(routes, { preloadingStrategy: PreloadAllModules })
  ],
  exports: [RouterModule]
})
export class AppRoutingModule { }

  args: string | null = null;

...
1 Es recomendable utilizar track para mejorar el rendimiento de la aplicación. Si no se usa track, Angular tratará a todos los elementos de la lista como nuevos en cada cambio y se volverán a renderizar completamente. Con track, Angular reutiliza los elementos que no han cambiado, lo que mejora el rendimiento. Sin track, si elements cambia, Angular borra y vuelve a insertar toda la lista. Con track, Angular solo actualiza los elementos que hayan cambiado.

1.3. Angular Router: Navigate

Además de RouterLink, Angular nos ofrece la posibilidad de navegar por las páginas haciendo uso de la operación navigate de Angular Router (ver enlace).

Para ello, añadimos un botón en la página inicial:

...

Además, en la definición del componente, realizamos las siguientes modificaciones:

Archivo home.page.ts

import { Router } from '@angular/router';

...

  constructor(private router: Router) {} (1)

  goToNewPageNavigate() {
    this.router.navigate(["/new-page", this.data]); (2)

  }

...
1 Inyección del Router
2 Uso del método navigate para acceder a la nueva página con el envío de la información de la propiedad data

En lugar de utilizar la función navigate, podemos hacer uso de navigateByUrl si queremos pasar argumento directamente la ruta a la que queremos acceder con el valor de la propiedad data como parte de la dirección:

goToNewPageNavigate() {
    this.router.navigateByUrl(`/new-page/${this.data}`);
}

2. Ejercicio: Creación de una aplicación Ionic con opciones de navegación

Como ejercicio, se debe crear una aplicación que tenga las siguientes características:

  • El inicio de la aplicación muestra tres pestañas. La que se abre por defecto es la segunda.

  • Las pestañas deben referirse a:

    • Clients

    • Reservations

    • Products

  • Los iconos de acceso a cada pestaña deben corresponderse con el tipo de información (clientes, productos y reservas).

  • El contenido de cada pestaña debe tener un botón que abra una nueva página.

  • Los botones de las tres pestañas deben aparecer exactamente a la misma altura, tres píxeles por debajo de la mitad del componente y centrados.

  • Las tres nuevas páginas a las que se accede, tienen similares características:

    • Muestran un listado de elementos de ejemplo (clientes, reservas y productos, respectivamente).

    • Tienen un botón para navegar hacia atrás (cada una debe volver a su pestaña)

  • La página de clientes tiene una funcionalidad adicional. Tiene un botón para ir a una nueva página para editar los datos del cliente. En dicha página no debe implementarse contenido, solo el título, pero permitirá dos cosas:

    • Volver atrás

    • Ir al inicio de la aplicación
